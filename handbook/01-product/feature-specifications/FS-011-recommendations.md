# Purpose

Define the personalised recommendation engine — providing students with contextually relevant suggestions for their next learning action based on adaptive profile, recent activity, and weak topics.

---

# Scope

- Personalised Lesson Recommendations
- Quiz Recommendations
- Revision Recommendations
- Document-Based Recommendations
- Recommendation Refresh

---

# Business Rules

1. Recommendations are generated by the AI Platform using the student's adaptive profile and recent activity.
2. Recommendations are cached in Redis with a 1-hour TTL.
3. Recommendations refresh automatically after a QuizCompleted or LessonCompleted event.
4. Students can manually request a recommendation refresh up to 5 times per hour.
5. Recommendations are personalised — different students receive different recommendations.
6. Each recommendation includes a type, topic, reason, priority, and difficulty.
7. Clicking a recommendation is tracked as a behaviour event.
8. Recommendations favour weak topics over strong topics.

---

# Recommendation Types

| Type | Description |
|------|-------------|
| lesson | Suggest generating a lesson on a specific topic |
| quiz | Suggest a quiz to test understanding |
| revision | Suggest revisiting a previously learned topic |
| document | Suggest asking the AI Tutor about an uploaded document |

---

# Recommendation Generation Logic

The AI Platform generates recommendations using:

- Current mastery score and level
- Weak topics list (prioritised)
- Strong topics list (deprioritised)
- Recent activity (last 5 lessons and quizzes)
- Learning goals from student profile
- Uploaded document topics

Output: ordered list of 5 recommendations with priority (high / medium / low).

---

# User Flow

```
Student opens dashboard or recommendations section
  ↓
GET /api/v1/recommendations
  ↓
Backend checks Redis cache
  ↓
If cache hit: return cached recommendations
  ↓
If cache miss: call AI Platform /recommendations/generate
  ↓
Store result in Redis (TTL: 1 hour)
  ↓
Return recommendations to frontend
  ↓
Student clicks a recommendation
  ↓
Behaviour event: recommendation_clicked recorded
  ↓
Frontend navigates to relevant action (lesson generation, quiz, etc.)
```

- [ ] Recommendations favour weak topics

- [ ] Clicking a recommendation records a behaviour event
