# Purpose

Define the AI-powered lesson generation feature — the core learning content engine of GenLearn.

Students request lessons on any topic and receive personalised, structured educational content generated by AI, optionally grounded in their uploaded documents.

---

# Scope

- Lesson Request
- Asynchronous Generation via AI Platform
- Adaptive Difficulty Personalisation
- RAG-Enhanced Content
- Lesson Storage and Retrieval
- Lesson Completion Tracking

---

# Business Rules

1. Lesson generation is always asynchronous.
2. Difficulty defaults to the student's current adaptive level if not specified.
3. When document IDs are provided, the lesson content is grounded in those documents via RAG.
4. Each generated lesson is stored permanently and can be revisited.
5. A student owns their lessons — other students cannot access them.
6. Lessons have a generation status: generating → ready → failed.
7. On completion, a LessonCompleted behaviour event is recorded.
8. Students are rate-limited to 20 lesson generations per hour.
9. A lesson cannot be submitted for completion if it has status: generating or failed.
10. Lessons cannot be edited — they are AI-generated and immutable.

---

# Lesson Content Structure

A complete generated lesson contains:

| Field | Description |
|-------|-------------|
| learningObjectives | What the student will understand after this lesson |
| explanation | Main content — clear, structured explanation |
| examples | Concrete worked examples |
| analogies | Real-world comparisons to aid understanding |
| keyPoints | Bulleted summary of essential concepts |
| summary | Short paragraph recap |
| revisionQuestions | 3–5 questions to test understanding |
| codeSnippets | Optional — for programming topics |

# User Flow: Lesson Generation

```
Student clicks "Generate Lesson"
  ↓
Enters topic (required) + optional difficulty and learning goal
  ↓
Optionally selects documents to include
  ↓
POST /api/v1/lessons/generate
  ↓
Backend creates lesson record (status: generating)
  ↓
Returns 202 with lessonId
  ↓
Frontend polls GET /api/v1/lessons/:id
  ↓
Backend queues AI Platform request via BullMQ
  ↓
AI Platform generates lesson content
  ↓
Backend stores content and updates status: ready
  ↓
Frontend receives ready lesson and renders content
```

---

# Adaptive Personalisation

The AI Platform receives the student's context with every lesson request:

- Current mastery level
- Adaptive score
- Weak and strong topics
- Learning goals
- Preferred difficulty

The AI adjusts:

- Explanation complexity
- Example difficulty
- Volume of prerequisite content
- Depth of coverage

---

# User Flow: Completing a Lesson

```
Student reads the lesson
  ↓
Clicks "Mark as Complete"
  ↓
POST /api/v1/lessons/:id/complete { timeSpentSeconds, rating }
  ↓
Backend records LessonCompleted behaviour event
  ↓
BullMQ triggers adaptive profile update
  ↓
Progress record updated
  ↓
Frontend shows completion confirmation and next recommendation
```

---

# Acceptance Criteria

- [ ] Student can request a lesson on any topic
- [ ] Generation returns 202 immediately
- [ ] Lesson status progresses: generating → ready
- [ ] Failed generation sets status: failed with an error message
- [ ] Lesson content includes all required sections
- [ ] Difficulty defaults to the student's adaptive level if not specified
- [ ] Document IDs can optionally be included to ground the lesson
- [ ] Student can mark a lesson as complete
- [ ] Completion records a behaviour event
- [ ] Student cannot access another student's lessons
- [ ] Rate limit of 20 generations per hour is enforced
